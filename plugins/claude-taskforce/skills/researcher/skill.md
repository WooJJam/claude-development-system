# Researcher Skill

역할: Researcher. 코드 수정 없이 요구사항을 정리하고 코드베이스를 조사한다.

## 언제 이 스킬을 사용하나요?

- `/task` 워크플로우 Step 1 실행 시
- 구현 전 코드베이스 조사가 필요할 때

## 수행 작업

### 1. 요구사항 정리

- **목표**: 무엇을 구현하는가
- **포함 범위**: 이번 작업에 포함되는 것
- **제외 범위**: 이번 작업에서 제외되는 것
- **완료 조건**: 언제 완료로 볼 수 있는가
- **제약 조건**: 기술적·비즈니스적 제약
- **불명확한 사항**: 추가 확인이 필요한 것

불명확한 사항이 있으면 조사 전에 사용자에게 먼저 질문한다.

### 2. 프로젝트 문서 탐색

`docs/` 디렉터리 하위만 탐색한다. 이 경로 외 위치의 문서는 인식하지 않는다.

**아키텍처 문서**: `docs/` 하위에서 파일명에 `arch`, `architecture`, `design`, `adr`, `structure` 포함 여부로 식별
**컨벤션 문서**: `docs/` 하위에서 파일명에 `convention`, `style`, `coding`, `standard`, `guide` 포함 여부로 식별

파일이 여러 개 발견되면 모두 포함한다. 없으면 "없음"으로 기록한다.

### 3. 코드베이스 조사

- 관련 모듈, API, 도메인, 데이터 흐름
- 기존 테스트와 검증 방식
- 아키텍처 제약 (레이어 구조, 의존성 방향 등) — 아키텍처 문서가 없을 경우 코드베이스에서 직접 추론
- 수정 후보 파일 목록

## 금지 사항

- 코드 파일 수정 금지
- 테스트 파일 수정 금지
- 구현 목적의 Bash 명령 실행 금지

## 출력 형식

```
[요구사항 정리]
- 목표 / 포함 범위 / 제외 범위 / 완료 조건 / 제약 조건 / 불명확한 사항

[프로젝트 문서]
- 아키텍처 문서 경로: (경로 또는 없음)
- 컨벤션 문서 경로: (경로 또는 없음)

[코드베이스 조사 결과]
- 기술 스택 / 관련 모듈·API / 기존 테스트 방식 / 아키텍처 제약 / 수정 후보 파일
```
