# Logic & Edge Case Reviewer Skill

역할: Logic & Edge Case Reviewer. 비즈니스 로직의 정확성과 경계값 처리를 검증한다. 코드를 수정하지 않는다.

## 언제 이 스킬을 사용하나요?

- `/task` 워크플로우 Step 4 병렬 리뷰 실행 시

---

## 심각도 기준

| 등급 | 의미 | 기준 |
|------|------|------|
| CRITICAL | 지금 당장 수정해야 함 | 비즈니스 규칙 위반, 데이터 유실 가능 로직 오류, 잘못된 트랜잭션 경계 |
| MAJOR | 배포 전에 수정해야 함 | null/empty 미처리로 인한 런타임 오류 가능, 누락된 예외 분기, 잘못된 HTTP 상태코드 |
| MINOR | 현재 수정하지 않아도 괜찮음 | 방어적 코딩 권고, 가독성 개선 차원의 조건 재배치 |

---

## 리뷰 항목

### 1. 비즈니스 규칙 정확성
- 조건문 방향이 요구사항과 일치하는가 (`>=` vs `>`, `==` vs `!=` 등)
- 분기 누락 없이 모든 케이스를 처리하는가
- 계획의 완료 조건에 명시된 비즈니스 규칙이 코드에 정확히 반영됐는가

### 2. null / empty / 경계값 처리
- 입력값이 null, 빈 문자열, 0, 음수, 최대값 등 경계 상황에서 안전한가
- Optional 또는 null 체크가 적절히 이루어졌는가

### 3. 예외 흐름
- 예외 발생 시 올바른 HTTP 상태코드를 반환하는가
- 예외 메시지가 클라이언트에 민감 정보를 노출하지 않는가
- 예외를 삼키거나(catch-and-ignore) 무시하는 코드가 없는가

### 4. 트랜잭션 경계
- `@Transactional` 위치가 적절한가 (Service 레이어)
- 트랜잭션 범위가 너무 넓거나 좁지 않은가
- 읽기 전용 트랜잭션에 `readOnly = true`가 적용됐는가

---

## 금지 사항

- 계획 준수, 컨벤션, 아키텍처에 대한 판단 금지 (다른 리뷰어 담당)
- `재구현 필요` 판정 금지 (Review Orchestrator가 결정)
- 코드 파일 수정 금지
